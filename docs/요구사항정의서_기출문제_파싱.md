## ⚙️ 3. 시스템 구성 단계

### 3.1 전체 파이프라인

```
Step 0: 웹 수집 (Tistory HTML → JSON 변환)
    ↓
Step 1: PDF Parsing (PyMuPDF / Cursor)
    ↓
Step 2: NLP 군집화 (KoSimCSE / OpenAI)
    ↓
Step 3: 빈출유형 통계
    ↓
Step 4: 핵심 해설 자동 생성
    ↓
Step 5: JSON/DB 구조화
    ↓
Step 6: 시각화 및 보고서 자동 출력
```

### 3.2 각 단계 상세

#### **Step 0: 웹 수집 (Tistory HTML → JSON)**
- Tistory 복원문제 HTML 크롤링
- BeautifulSoup으로 본문 추출
- 문제 단위 분리 및 JSON 저장
- 출력: `data/raw/tistory_questions_YYYY_roundX.json`

#### **Step 1: PDF Parsing**
- PyMuPDF/pdfplumber로 PDF 텍스트 추출
- 문제번호, 선택지, 정답, 해설 분리
- 이미지/표 캡처 및 저장
- 출력: `data/raw/pdf_questions_YYYY_roundX.json`

#### **Step 2: NLP 군집화**
- SentenceTransformer(KoSimCSE) 임베딩 생성
- 코사인 유사도 기반 문제 군집화
- 유사 문제 그룹 생성
- 출력: `data/processed/clusters.json`

#### **Step 3: 빈출유형 통계**
- 6개년 데이터 통계 분석
- 유형별/난이도별 출제 빈도 계산
- 트렌드 분석 (증가/감소 추세)
- 출력: `data/stats/frequency_report.json`

#### **Step 4: 핵심 해설 자동 생성**
- OpenAI API로 고등학생 수준 해설 생성
- 키워드 자동 추출
- 학습 스킬 태그 부여
- 출력: `data/processed/questions_with_explanation.json`

#### **Step 5: JSON/DB 구조화**
- 표준 스키마로 변환
- JSONL 파일 생성
- SQLite DB 저장 (선택)
- 출력: `data/final/questions_2020_2025.jsonl`

#### **Step 6: 시각화 및 보고서**
- Chart.js로 통계 시각화
- PDF 보고서 생성
- 대시보드 업데이트

## 📘 4. Step 0 — 온라인 복원문제 데이터 수집

### 4.1 웹 크롤링 스크립트

chobopark.tistory.com의 2020~2025년 본문 데이터를 수집하여 JSON 형태로 저장합니다.

### 4.2 수집 스크립트 예시 (Python)

```python
import requests
import re
import json
from bs4 import BeautifulSoup

# 2020~2025년 전체 URL 목록
urls = [
    # 2020년
    "https://chobopark.tistory.com/401",
    "https://chobopark.tistory.com/403",
    "https://chobopark.tistory.com/404",
    # 2021년
    "https://chobopark.tistory.com/420",
    "https://chobopark.tistory.com/422",
    "https://chobopark.tistory.com/425",
    # 2022년
    "https://chobopark.tistory.com/463",
    "https://chobopark.tistory.com/482",
    "https://chobopark.tistory.com/494",
    # 2023년
    "https://chobopark.tistory.com/516",
    "https://chobopark.tistory.com/528",
    "https://chobopark.tistory.com/538",
    # 2024년
    "https://chobopark.tistory.com/546",
    "https://chobopark.tistory.com/552",
    "https://chobopark.tistory.com/553",
    # 2025년
    "https://chobopark.tistory.com/554",
    "https://chobopark.tistory.com/555"
]

def extract_questions(url):
    """Tistory 페이지에서 문제 추출"""
    html = requests.get(url).text
    soup = BeautifulSoup(html, "html.parser")
    text = soup.get_text("\n")

    # 문제 단위 분리
    questions = re.split(r"\n\d+\.", text)
    results = []
    
    for q in questions:
        if len(q.strip()) > 30:  # 최소 길이 필터
            item = {
                "source": url,
                "question": q.split("①")[0].strip() if "①" in q else q.strip(),
                "options": re.findall(r"[①②③④].*", q),
                "answer": re.findall(r"정답[:：]?\s*[①②③④A-Dㄱ-ㄷ1-4]", q),
                "explanation": re.findall(r"(해설|풀이|설명)[:：]?\s*(.*)", q, re.DOTALL)
            }
            results.append(item)
    
    return results

# 전체 데이터 수집
all_data = []
for u in urls:
    print(f"수집 중: {u}")
    try:
        data = extract_questions(u)
        all_data.extend(data)
        print(f"✅ {len(data)}개 문제 추출")
    except Exception as e:
        print(f"❌ 오류: {e}")

# JSON 파일 저장
with open("data/raw/tistory_questions_2020_2025.json", "w", encoding="utf-8") as f:
    json.dump(all_data, f, ensure_ascii=False, indent=2)

print(f"\n총 {len(all_data)}개 문제 수집 완료!")
```

### 4.3 출력 결과

➡️ 이 스크립트를 실행하면 `data/raw/tistory_questions_2020_2025.json` 파일이 생성되어 ChatGPT / Cursor / Notion 학습 파이프라인에 바로 업로드 가능합니다.

  - 임베딩 기반: 0.4

### 6.3 3차 처리: 다중 라벨 허용 (혼합형)

- **JOIN + 집계** 동시 요구 등 혼합형은 `type_sub: ["A2","A3"]` 같이 배열 허용
- **UI 노출**: 대표유형 하나(우선순위 높은 것)로 노출, 상세에서 보조유형 표시

## 🔄 8. 파이프라인 (2020~2025 일괄 적용)

### 8.1 데이터 수집 단계

```
1. 수집: 회차 URL 목록(2020~2025, 401…555) → HTML 본문 텍스트화
   - requests + BeautifulSoup
   - HTML → JSON 변환
   - 출력: data/raw/tistory_YYYY_roundX.json

2. 문항 분리:
   - 문제번호 패턴: (?m)^\s*\d+\.\s+
   - 선택지: (?m)^[①②③④].+$ 또는 [1-4]\)
   - 정답: (?i)정답\s*[:：]?\s*([①②③④A-Dㄱ-ㄷ1-4])
   - 해설: (?i)(해설|풀이|설명)[:：]?\s*(.*)

3. 정답 정규화:
   ①②③④ → 1/2/3/4
   A-D → 1/2/3/4
   ㄱㄴㄷㄹ → 1/2/3/4
   1-4 → 1/2/3/4
```

### 8.2 분류 및 라벨링 단계

```
4. 분류: 위 라벨링 규칙 적용
   - 규칙 기반 1차 분류
   - 임베딩 기반 2차 보정
   - 다중 라벨 허용

5. 저장:
   - questions_YYYY_roundX.json (회차별)
   - questions_2020_2025.jsonl (전체 합본)

6. 색인:
   - SQLite DB 저장 (선택)
   - Notion 업로드 (선택)
```

### 8.3 학습모드 노출

```
7. 회차별 모드:
   - year, round로 필터 → q_no 오름차순

8. 유형별 모드:
   - type_sub 포함 여부로 필터 (예: A2 = JOIN 모음)
```

## 📝 16. 추가 고려사항

### 16.1 PDF vs 웹 데이터
- **PDF**: 공식 기출문제 (정확도 높음, 하지만 OCR 필요 가능)
- **웹**: 복원문제 (접근 쉬움, 해설 포함, 하지만 오타 가능)
- **권장**: 웹 데이터 우선 수집 후, PDF로 교차 검증

### 16.2 OCR 품질 개선
- 스캔 PDF의 경우 Tesseract OCR 사용
- 정확도 낮으면 Google Vision API / AWS Textract 고려
- OCR 결과는 항상 수동 검수 필요

### 16.3 이미지 캡처 자동화
- PDF 렌더링 시 이미지 자동 추출
- 실패 시 수동 스크린샷으로 보완
- 파일 명명 규칙: `{year}_r{round}_q{q_no}.png`
